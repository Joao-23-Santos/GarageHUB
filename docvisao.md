# Documento de Visão — GarageHUB

## (a) Objetivo — O que o sistema fará?

O GarageHUB é um sistema de marketplace que tem como objetivo centralizar e facilitar a administração de serviços automóveis, clientes e veículos.

O sistema permitirá:
- Registar clientes e veículos
- Gerir anuncios de veículos
- Melhorar a comunicação entre vendedor e cliente

---

## (b) Escopo — Onde pode ser usado?

O sistema poderá ser utilizado em:
- android

---

## (c) Partes Interessadas (Stakeholders)

### Utilizadores principais:
- Vendedores 
- Compradores

### Beneficiários:
- Clientes (melhor exposição do anuncio)
- Proprietários dos Veículos (melhor controlo e eficiência)

## (f) Arquitetura de Referência

### Componentes principais:
- Frontend (interface do utilizador)
- Backend (API / lógica de negócio)
- Base de dados (armazenamento de dados)

### Exemplo de arquitetura:

[Frontend] → [API Backend] → [Base de Dados]

Possível stack:
- Frontend: flutter
- Backend: Node.js 
- Base de dados: Supabase

---

## (g) Restrições do Produto

- Dependência de ligação à internet 
- Limitações de performance com muitos dados
- Compatibilidade com diferentes dispositivos
- Possíveis erros de input por parte do utilizador

---

## Conclusão

O GarageHUB visa modernizar a gestão de veículos, aumentando a eficiência operacional, melhorando a experiência do cliente e centralizando toda a informação relevante num único sistema.
